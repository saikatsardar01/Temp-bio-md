Greedy Algorithm

1. Fractional Knapsack — A Small-Scale Juice Bar

Sarah's cooler has capacity 15 kg. Each fruit can be taken in any fraction, so the Fractional Knapsack greedy strategy applies: always take as much as possible of the fruit with the highest profit-per-kg first.

FruitAvailable (kg)Profit/kg (INR)
Strawberries8250
Blueberries5380
Raspberries6320
Mangoes10180
Kiwis4290

Algorithm

  1. Compute ratioi = profit_per_kg for every fruit (already given per kg, so no extra division needed).
  2. Sort all fruits in decreasing order of profit/kg → Blueberries(380) > Kiwis(290) > Raspberries(320)… (sorted: Blueberries, Raspberries, Kiwis, Strawberries, Mangoes).
  3. Initialise remaining capacity W = 15, totalProfit = 0.
  4. For each fruit in sorted order:
    • If its full weight ≤ remaining capacity → take it entirely, subtract weight from W, add weight×ratio to profit.
    • Else → take only the fraction that fits (remaining/weight), add fraction×weight×ratio to profit, set W = 0, stop.
  5. Return total profit.

Result for this instance: take Blueberries 5 kg (1900), Raspberries 6 kg (1920), Kiwis 4 kg (1160) → 15 kg filled → total profit = 4980 INR (Strawberries/Mangoes are never touched since capacity is exhausted).

C and Python Implementation

C
#include <stdio.h>
#include <stdlib.h>

typedef struct {
    char name[20];
    double weight;
    double profitPerKg;
} Fruit;

int compare(const void *a, const void *b) {
    Fruit *f1 = (Fruit *)a;
    Fruit *f2 = (Fruit *)b;
    if (f2->profitPerKg - f1->profitPerKg > 0) return 1;
    if (f2->profitPerKg - f1->profitPerKg < 0) return -1;
    return 0;
}

int main() {
    Fruit fruits[] = {
        {"Strawberries", 8, 250},
        {"Blueberries",  5, 380},
        {"Raspberries",  6, 320},
        {"Mangoes",     10, 180},
        {"Kiwis",        4, 290}
    };
    int n = 5;
    double capacity = 15.0;

    qsort(fruits, n, sizeof(Fruit), compare);   /* sort by profit/kg desc */

    double totalProfit = 0.0;
    printf("Item Selection:\n");
    for (int i = 0; i < n && capacity > 0; i++) {
        if (fruits[i].weight <= capacity) {
            capacity -= fruits[i].weight;
            totalProfit += fruits[i].weight * fruits[i].profitPerKg;
            printf("Take %.2f kg of %s (full)\n", fruits[i].weight, fruits[i].name);
        } else {
            totalProfit += capacity * fruits[i].profitPerKg;
            printf("Take %.2f kg of %s (partial)\n", capacity, fruits[i].name);
            capacity = 0;
        }
    }
    printf("Maximum Profit = %.2f INR\n", totalProfit);
    return 0;
}
Python
def fractional_knapsack(capacity, items):
    # items: list of (name, weight, profit_per_kg)
    items.sort(key=lambda x: x[2], reverse=True)   # sort by ratio desc
    total_profit = 0.0
    remaining = capacity
    selection = []

    for name, weight, ppk in items:
        if remaining <= 0:
            break
        if weight <= remaining:
            remaining -= weight
            total_profit += weight * ppk
            selection.append((name, weight, "full"))
        else:
            total_profit += remaining * ppk
            selection.append((name, remaining, "partial"))
            remaining = 0

    return total_profit, selection

        ("Blueberries",  5, 380),
        ("Raspberries",  6, 320),
        ("Mangoes",     10, 180),
        ("Kiwis",        4, 290),
    ]
    capacity = 15
    profit, chosen = fractional_knapsack(capacity, fruits)
    for name, w, kind in chosen:
        print(f"Take {w:.2f} kg of {name} ({kind})")
    print(f"Maximum Profit = {profit:.2f} INR")

2. Job Sequencing with Deadlines — The Bakery's Daily Rush

Only one oven (single processor), each order takes one time unit (one hour slot), and a job is only useful if completed by its deadline. This is the classic Job Sequencing with Deadlines problem.

OrderDeadline (h)Profit (INR)
C123000
C212000
C344500
C431000
C523500
C652500
C731500

Algorithm

  1. Sort all orders in decreasing order of profit.
  2. Let D = maximum deadline among all jobs; create a slot array slot[1..D], all free.
  3. For each order (highest profit first): scan slots from min(its deadline, D) down to 1; place the order in the first free slot found (i.e., latest slot ≤ deadline). If found, mark slot occupied and add profit.
  4. If no free slot ≤ deadline exists, the order is skipped (rejected).
  5. The filled slots give the optimal schedule; sum of their profits is the maximum total profit.

Result for this instance (sorted by profit: C3-4500, C5-3500, C1-3000, C6-2500, C2-2000, C7-1500, C4-1000): Hour1→C1, Hour2→C5, Hour3→C7, Hour4→C3, Hour5→C6 → total profit = 3000+3500+1500+4500+2500 = 15000 INR (C2 and C4 are rejected since no free slot remains at or before their deadlines).

C and Python Implementation

        if (jobs[i].deadline > maxDeadline) maxDeadline = jobs[i].deadline;

    int *slot = (int *)calloc(maxDeadline + 1, sizeof(int));
    char slotJob[10][5];

    for (int i = 0; i < n; i++) {
        int d = jobs[i].deadline;
        if (d > maxDeadline) d = maxDeadline;
        for (int t = d; t >= 1; t--) {           /* latest free slot <= deadline */
            if (slot[t] == 0) {
                slot[t] = 1;
                strcpy(slotJob[t], jobs[i].id);
                totalProfit += jobs[i].profit;
                break;
            }
        }
    }

    printf("Scheduled Orders:\n");
    for (int t = 1; t <= maxDeadline; t++)
        if (slot[t]) printf("Hour slot %d -> %s\n", t, slotJob[t]);
    printf("Maximum Profit = %d INR\n", totalProfit);

    free(slot);
    return 0;
}
Python
def job_sequencing(jobs):
    # jobs: list of (id, deadline, profit)
    jobs = sorted(jobs, key=lambda x: x[2], reverse=True)
    max_deadline = max(j[1] for j in jobs)
    slot = [None] * (max_deadline + 1)
    total_profit = 0

    for jid, deadline, profit in jobs:
        d = min(deadline, max_deadline)
        for t in range(d, 0, -1):        # latest free slot <= deadline
            if slot[t] is None:
                slot[t] = jid
                total_profit += profit
                break

    schedule = [(t, slot[t]) for t in range(1, max_deadline + 1) if slot[t]]
    return schedule, total_profit

        ("C6", 5, 2500),

3. Binary Search — Finding a Product in a Large E-commerce Catalog

The catalog is already sorted alphabetically and we know the exact target name ("XtremeGamer Pro 2025"), so Binary Search is the efficient choice instead of a linear scan.

Algorithm

  1. Set low = 0, high = n−1 (n = number of products).
  2. While low ≤ high:
    • mid = (low + high) / 2
    • Compare catalog[mid] with the target name (lexicographic string comparison).
    • If equal → product found at index mid, stop.
    • If catalog[mid] < target → search right half: low = mid + 1.
    • Else → search left half: high = mid − 1.
  3. If the loop ends without a match, the product is not in the catalog.

C and Python Implementation

C
#include <stdio.h>
#include <string.h>

int binarySearch(char *arr[], int n, char *target) {
    int low = 0, high = n - 1;
    while (low <= high) {
        int mid = low + (high - low) / 2;
        int cmp = strcmp(arr[mid], target);
        if (cmp == 0) return mid;
        else if (cmp < 0) low = mid + 1;
        else high = mid - 1;
    }
    return -1;
}

4. Merge Sort — Consolidating and Organizing Student Exam Scores

Two unsorted lists (150 scores from Section A, 120 scores from Section B) must be combined into one ascending list of 270 scores. Merge Sort is ideal: it gives guaranteed O(n log n) performance regardless of the input order, and its core "merge" step is exactly the operation needed to combine two lists.

Algorithm

  1. Concatenate Section A and Section B scores into one array of size n = 150 + 120 = 270 (O(n)).
  2. Recursively divide the array into two halves until each sub-array has ≤ 1 element.
  3. Recursively sort each half (Merge Sort on the halves).
  4. Merge the two sorted halves back together: repeatedly compare the front elements of both halves and append the smaller one to the result, until both halves are exhausted.
  5. The final merged array is the fully sorted combined list of all 270 scores — from which min, max, and median can be read directly, and grade bands can be computed with a single linear pass.

C and Python Implementation

C
#include <stdio.h>
#include <stdlib.h>

void merge(int arr[], int l, int m, int r) {
    int n1 = m - l + 1, n2 = r - m;
    int *L = malloc(n1 * sizeof(int));
    int *R = malloc(n2 * sizeof(int));
    for (int i = 0; i < n1; i++) L[i] = arr[l + i];
    for (int j = 0; j < n2; j++) R[j] = arr[m + 1 + j];

    int i = 0, j = 0, k = l;
    while (i < n1 && j < n2)
        arr[k++] = (L[i] <= R[j]) ? L[i++] : R[j++];
    while (i < n1) arr[k++] = L[i++];
    while (j < n2) arr[k++] = R[j++];

    free(L); free(R);
}

void mergeSort(int arr[], int l, int r) {
    if (l < r) {
        int m = l + (r - l) / 2;
        mergeSort(arr, l, m);
        mergeSort(arr, m + 1, r);
        merge(arr, l, m, r);
    }
}

int main() {
    /* Section A (150 scores) + Section B (120 scores) -> combined = 270 scores.
       Small illustrative subset shown here; in practice load both spreadsheets
       into one array of size 270 before sorting. */
    int sectionA[] = {88, 45, 76, 90, 60};
    int sectionB[] = {55, 99, 70, 40};
    int nA = 5, nB = 4, n = nA + nB;

    int *combined = malloc(n * sizeof(int));
    for (int i = 0; i < nA; i++) combined[i] = sectionA[i];
    for (int i = 0; i < nB; i++) combined[nA + i] = sectionB[i];

    mergeSort(combined, 0, n - 1);

    printf("Combined sorted scores:\n");
    for (int i = 0; i < n; i++) printf("%d ", combined[i]);
    printf("\nMinimum = %d, Maximum = %d\n", combined[0], combined[n - 1]);

    free(combined);
    return 0;
}
Python
def merge_sort(arr):
    if len(arr) <= 1:
        return arr
    mid = len(arr) // 2
    left = merge_sort(arr[:mid])
    right = merge_sort(arr[mid:])
    return merge(left, right)


def merge(left, right):
    result, i, j = [], 0, 0
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:
            result.append(left[i]); i += 1
        else:
            result.append(right[j]); j += 1
    result.extend(left[i:])
    result.extend(right[j:])
    return result


if __name__ == "__main__":
    section_a = [88, 45, 76, 90, 60]   # subset representing 150 scores
    section_b = [55, 99, 70, 40]       # subset representing 120 scores
    combined = section_a + section_b
    sorted_scores = merge_sort(combined)
    print("Combined sorted scores:", sorted_scores)
    print(f"Minimum = {sorted_scores[0]}, Maximum = {sorted_scores[-1]}")
    n = len(sorted_scores)
    median = (sorted_scores[n // 2] if n % 2 else
              (sorted_scores[n // 2 - 1] + sorted_scores[n // 2]) / 2)
    print(f"Median = {median}")

5. Dijkstra's Algorithm — Optimizing Delivery Routes for a Local Pharmacy

Mr. Das needs the shortest travel time from a single source (Pharmacy P) to every customer/intersection. All road travel times are positive, so Dijkstra's Algorithm applies directly.

Algorithm

  1. Model the road network as a weighted directed graph: vertices = {P, I1, I2, I3, C1, C2, C3, C4}, edges = roads with travel-time weights.
  2. Initialise dist[P] = 0 and dist[v] = ∞ for every other vertex; mark all vertices unvisited.
  3. Repeat until all vertices are visited:
    • Pick the unvisited vertex u with the smallest dist[u] (via a min-priority-queue / min-heap).
    • Mark u visited.
    • For every edge (u → v) with weight w: relax — if dist[u] + w < dist[v], update dist[v] = dist[u] + w.
  4. dist[] now holds the shortest travel time from P to every location.

C and Python Implementation

C
#include <stdio.h>
#include <limits.h>

#define V 8
#define INF INT_MAX

int minDistance(int dist[], int visited[]) {
    int min = INF, minIndex = -1;
    for (int v = 0; v < V; v++)
        if (!visited[v] && dist[v] <= min) { min = dist[v]; minIndex = v; }
    return minIndex;
}

void dijkstra(int graph[V][V], int src, char *names[]) {
    int dist[V], visited[V] = {0};
    for (int i = 0; i < V; i++) dist[i] = INF;
    dist[src] = 0;

    for (int count = 0; count < V - 1; count++) {
        int u = minDistance(dist, visited);
        if (u == -1) break;
        visited[u] = 1;
        for (int v = 0; v < V; v++)
            if (!visited[v] && graph[u][v] && dist[u] != INF &&
                dist[u] + graph[u][v] < dist[v])
                dist[v] = dist[u] + graph[u][v];
    }

    printf("Shortest travel time from %s:\n", names[src]);
    for (int i = 0; i < V; i++)
        printf("  to %s: %d min\n", names[i], dist[i]);
}

int main() {
    char *names[V] = {"P", "I1", "I2", "I3", "C1", "C2", "C3", "C4"};
    int graph[V][V] = {0};
    /* indices: P=0 I1=1 I2=2 I3=3 C1=4 C2=5 C3=6 C4=7 */
    graph[0][1] = 5;  graph[0][2] = 7;
    graph[1][4] = 3;  graph[1][5] = 6;  graph[1][3] = 4;

    return 0;
}
Python
import heapq

def dijkstra(graph, src):
    dist = {v: float('inf') for v in graph}
    dist[src] = 0
    pq = [(0, src)]
    visited = set()

    while pq:
        d, u = heapq.heappop(pq)
        if u in visited:
            continue
        visited.add(u)
        for v, w in graph[u]:
            if d + w < dist[v]:
                dist[v] = d + w
                heapq.heappush(pq, (dist[v], v))
    return dist

        "P":  [("I1", 5), ("I2", 7)],
        "I1": [("C1", 3), ("C2", 6), ("I3", 4)],
        "I2": [("C2", 4), ("C4", 3), ("I3", 2)],
        "I3": [("I2", 2), ("C3", 5), ("I1", 5)],
        "C1": [("I1", 4)],
        "C2": [("I2", 3)],
        "C3": [("P", 15)],
        "C4": [("I2", 2)],
    }
    distances = dijkstra(graph, "P")
    for loc, d in distances.items():

6. Floyd–Warshall Algorithm — Optimizing Inter-Branch Logistics

Mr. Roy needs the shortest travel time between every pair of branches/warehouse, computed once and reused for any query — this is the All-Pairs Shortest Path problem, solved efficiently with the Floyd–Warshall dynamic-programming algorithm (works even with mixed, non-uniform directed weights). Note: the problem statement does not supply an explicit graph for this scenario (only example queries), so a representative 5-location example (Warehouse + 4 branches) is used below to demonstrate the algorithm and code — the same code works unchanged on the real MegaMart travel-time matrix.

Algorithm

  1. Number the n locations 1..n. Build an n×n matrix dist[][] where dist[i][j] = direct travel time from i to j if a road exists, dist[i][i] = 0, and dist[i][j] = ∞ if no direct road exists.
  2. For each intermediate vertex k = 1 to n:
    • For each pair (i, j): if dist[i][k] + dist[k][j] < dist[i][j], update dist[i][j] = dist[i][k] + dist[k][j] — i.e., check whether routing through k gives a shorter path.
  3. After considering all n intermediate vertices, dist[i][j] holds the shortest travel time between every pair (i, j).
  4. The largest finite value in the resulting matrix identifies the two branches that are furthest apart (Mr. Roy's third question).

C and Python Implementation

C
#include <stdio.h>

#define V 5
#define INF 99999

void floydWarshall(int graph[V][V], char *names[]) {
    int dist[V][V];
    for (int i = 0; i < V; i++)
        for (int j = 0; j < V; j++)
            dist[i][j] = graph[i][j];

    for (int k = 0; k < V; k++)
        for (int i = 0; i < V; i++)
            for (int j = 0; j < V; j++)
                if (dist[i][k] + dist[k][j] < dist[i][j])
                    dist[i][j] = dist[i][k] + dist[k][j];

    printf("All-pairs shortest travel times (minutes):\n");
    printf("%10s", "");
    for (int j = 0; j < V; j++) printf("%10s", names[j]);
    printf("\n");
    for (int i = 0; i < V; i++) {
        printf("%10s", names[i]);
        for (int j = 0; j < V; j++) {
            if (dist[i][j] >= INF) printf("%10s", "INF");
            else printf("%10d", dist[i][j]);
        }
        printf("\n");
    }
}

Floyd–Warshall is preferred here over running Dijkstra from every vertex (O(V · (V+E) log V)) because it is simpler to implement, works uniformly on a dense adjacency matrix, and computing the whole table once lets Mr. Roy answer any future pairwise query in O(1) lookup time.

Backtracking

7. N-Queens (Backtracking) — Strategic Placement of Emergency Service Hubs

Placing 6 hubs on a 6×6 grid such that no two share a row, column, or diagonal is exactly the classic N-Queens problem (N = 6). The urban planners want all valid configurations, so a full backtracking search (rather than stopping at the first solution) is used.

Algorithm

  1. Represent the placement as board[row] = column, i.e., process the grid row by row and place exactly one hub per row (guarantees the row constraint automatically).
  2. For row = 0: try every column 0..N−1.
  3. A placement at (row, col) is safe if no previously placed hub shares:
    • the same column: board[i] == col,
    • the same main diagonal: board[i] − i == col − row,
    • the same anti-diagonal: board[i] + i == col + row, for all earlier rows i.
  4. If safe, place the hub (board[row] = col) and recurse to row + 1.
  5. If row == N, a complete valid configuration has been found — record/print it.
  6. Backtrack: after exploring a column choice (or if no column in the current row is safe), remove the hub and try the next column; if all columns are exhausted, backtrack to the previous row.
  7. Continue until every possibility has been explored, collecting every valid configuration.

For N = 6 there are exactly 4 valid configurations (a well-known result of the 6-Queens problem).

C and Python Implementation

C
#include <stdio.h>
#include <stdbool.h>

#define N 6
int board[N];
int solutionCount = 0;

bool isSafe(int row, int col) {
    for (int i = 0; i < row; i++) {
        if (board[i] == col ||
            board[i] - i == col - row ||
            board[i] + i == col + row)
            return false;
    }
    return true;
}

void solve(int row) {
    if (row == N) {
        solutionCount++;
        printf("Solution %d: ", solutionCount);
        for (int i = 0; i < N; i++) printf("(row %d, col %d) ", i, board[i]);
        printf("\n");
        return;
    }
    for (int col = 0; col < N; col++) {
        if (isSafe(row, col)) {
            board[row] = col;
            solve(row + 1);
        }
    }
}

int main() {
    solve(0);
    printf("Total valid configurations for %d hubs on %dx%d grid: %d\n",
           N, N, N, solutionCount);
    return 0;
}
Python
N = 6

def is_safe(board, row, col):
    for i in range(row):
        if board[i] == col or abs(board[i] - col) == row - i:
            return False
    return True

def solve(board, row, solutions):
    if row == N:
        solutions.append(board[:])
        return
    for col in range(N):
        if is_safe(board, row, col):
            board[row] = col
            solve(board, row + 1, solutions)
            board[row] = -1

if __name__ == "__main__":
    board = [-1] * N
    solutions = []
    solve(board, 0, solutions)
    for idx, sol in enumerate(solutions, 1):
        print(f"Solution {idx}: {[(r, c) for r, c in enumerate(sol)]}")
    print(f"Total valid configurations for {N} hubs on {N}x{N} grid: {len(solutions)}")

Dynamic Programming

8. Matrix Chain Multiplication — Optimizing Data Transformation Workflow

The fixed pipeline N → F → D → C is a chain of 4 matrix multiplications M1×M2×M3×M4. Since matrix multiplication is associative, the grouping (parenthesization) affects the total scalar multiplication cost — solved optimally with the Matrix Chain Multiplication (MCM) dynamic-programming algorithm.

MatrixDimensions
M1 (Normalization)2000 × 10000
M2 (Feature Selection)10000 × 5000
M3 (Dimensionality Reduction)5000 × 500
M4 (Clustering Analysis)500 × 50

Dimension array p = [2000, 10000, 5000, 500, 50] (matrix Mi has dimensions p[i−1] × p[i]).

Algorithm

  1. Let dp[i][j] = minimum scalar multiplications needed to compute the product Mi×…×Mj.
  2. Base case: dp[i][i] = 0 (a single matrix needs no multiplication).
  3. For chain length L = 2 to n (n = number of matrices = 4):
    • For every starting index i where j = i + L − 1 ≤ n:
      • dp[i][j] = min over all split points k (i ≤ k < j) of: dp[i][k] + dp[k+1][j] + p[i−1]×p[k]×p[j]
      • Record the best split point k in split[i][j] (for reconstructing the parenthesization).
  4. dp[1][n] holds the minimum total scalar multiplications for the whole chain.
  5. Recursively reconstruct the optimal grouping from the split[][] table.

Result for this instance (as computed by the DP table / program above): the optimal grouping is (M1×(M2×(M3×M4))) — i.e. first multiply M3×M4, then multiply that result by M2, and finally by M1 — giving a minimum cost of 3,625,000,000 scalar multiplications, far fewer than the naïve left-to-right grouping ((M1×M2)×M3)×M4, which costs 100,000,000,000 + 5,000,000,000 + 50,000,000 = 105,050,000,000 multiplications.

C and Python Implementation

C
#include <stdio.h>

#define N 5   /* p[] has n+1 = 5 elements for 4 matrices */

/* long long is required: costs here exceed 2^31 and overflow a plain int */
long long dp[N][N];
int split[N][N];

void printOptimalOrder(int i, int j, char *names[]) {
    if (i == j) {
        printf("%s", names[i - 1]);
        return;
    }
    printf("(");
    printOptimalOrder(i, split[i][j], names);
    printOptimalOrder(split[i][j] + 1, j, names);
    printf(")");
}

int main() {
    long long p[] = {2000, 10000, 5000, 500, 50};   /* dims for M1..M4 */
    char *names[] = {"M1", "M2", "M3", "M4"};
    int n = 4;   /* number of matrices */

    for (int i = 1; i <= n; i++) dp[i][i] = 0;

    for (int len = 2; len <= n; len++) {
        for (int i = 1; i <= n - len + 1; i++) {
            int j = i + len - 1;
            dp[i][j] = -1;
            for (int k = i; k < j; k++) {
                long long cost = dp[i][k] + dp[k + 1][j] + p[i - 1] * p[k] * p[j];
                if (dp[i][j] == -1 || cost < dp[i][j]) {
                    dp[i][j] = cost;
                    split[i][j] = k;
                }
            }
        }
    }

    printf("Minimum scalar multiplications = %lld\n", dp[1][n]);
    printf("Optimal parenthesization: ");
    printOptimalOrder(1, n, names);
    printf("\n");
    return 0;
}
Python
def matrix_chain_order(p):
    n = len(p) - 1  # number of matrices
    dp = [[0] * (n + 1) for _ in range(n + 1)]
    split = [[0] * (n + 1) for _ in range(n + 1)]

    for length in range(2, n + 1):
        for i in range(1, n - length + 2):
            j = i + length - 1
            dp[i][j] = float('inf')
            for k in range(i, j):
                cost = dp[i][k] + dp[k + 1][j] + p[i - 1] * p[k] * p[j]
                if cost < dp[i][j]:
                    dp[i][j] = cost
                    split[i][j] = k
    return dp, split


def print_optimal_order(split, i, j, names):
    if i == j:
        return names[i - 1]
    left = print_optimal_order(split, i, split[i][j], names)
    right = print_optimal_order(split, split[i][j] + 1, j, names)
    return f"({left}{right})"


if __name__ == "__main__":
    p = [2000, 10000, 5000, 500, 50]
    names = ["M1", "M2", "M3", "M4"]
    dp, split = matrix_chain_order(p)
    n = len(names)
    print(f"Minimum scalar multiplications = {dp[1][n]}")
    print("Optimal parenthesization:", print_optimal_order(split, 1, n, names))
